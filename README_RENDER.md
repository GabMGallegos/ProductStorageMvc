# ProductStorageMvc - Docker deployment on Render

This project is an ASP.NET Core MVC application with Entity Framework Core and Supabase PostgreSQL.
It does not use a REST API. The application renders Razor views and connects directly to PostgreSQL through Npgsql.

## Required Render environment variable

Create this environment variable in Render:

```text
ConnectionStrings__DefaultConnection
```

Example value:

```text
Host=aws-1-us-west-1.pooler.supabase.com;Port=5432;Database=postgres;Username=postgres.YOUR_PROJECT_REF;Password=YOUR_DATABASE_PASSWORD;SSL Mode=Require;Trust Server Certificate=true;Timeout=30;Command Timeout=60
```

Use the Session Pooler connection from Supabase, preferably port 5432.
Do not commit the real password to GitHub.

## Supabase table

If the table does not exist, run this SQL in the Supabase SQL Editor:

```sql
create table if not exists products (
    id integer generated by default as identity primary key,
    name varchar(100) not null,
    price numeric(10, 2) not null,
    quantity integer not null
);
```

## Local Docker test

```bash
docker build -t product-storage-mvc .
docker run --rm -p 10000:10000 -e "ConnectionStrings__DefaultConnection=YOUR_CONNECTION_STRING" product-storage-mvc
```

Open:

```text
http://localhost:10000
```

## Render deployment steps

1. Upload this project to GitHub.
2. In Render, create a new Web Service.
3. Select the GitHub repository.
4. Select Docker as the runtime.
5. Make sure Dockerfile Path is `Dockerfile`.
6. Add the environment variable `ConnectionStrings__DefaultConnection`.
7. Deploy.
